Filling the grid

Teodor Maxim

Six years ago while building a goofy side project I faced the following problem: how do I fit 48 squares on a screen so that they’re as big as possible? I had just started high school and was programming to procrastinate on my math homework, hence I shamelessly copied a (partly wrong) solution off of StackOverflow and moved on.

There was only code – no correctness proof, no complexity analysis. Having just repeated and passed the first Basisprüfungsblock (finally), I learned with pain that this does not fly at ETH. Using my new hard earned knowledge I set out to make them myself, procrastinating this time on my math homework with… math.

Let's state the problem: find a b×a∈Nb \times a \in \N grid (rows x columns) into a w×h∈R+w \times h \in \R^{+} rectangle which fits n∈Nn \in \N squares such that the side length of the squares s∈R+s \in \R^{+} is maximized and the grid does not overflow the rectangle. Since the side length is maximized, the grid will fit in the rectangle's full width or height, or both if a perfect fit is possible. Formally:

sw≔max⁡{ s∈R+  |  ∃a,b∈N . n≤ab∧as=w∧bs≤h }s_w \coloneqq \max \Set{ s \in \R^{+} | \exist a, b \in \N \text{ . } n \le ab \land as = w \land bs \le h } sh≔max⁡{ s∈R+  |  ∃a,b∈N . n≤ab∧as≤h∧bs=h }s_h \coloneqq \max \Set{ s \in \R^{+} | \exist a, b \in \N \text{ . } n \le ab \land as \le h \land bs = h } s≔max⁡{ sw,sh }.s \coloneqq \max \Set{ s_w, s_h }.

sws_w is the fit-width solution, shs_h the fit-height solution. The final solution ss is the desired maximal side length. Our target is to build an algorithm to find ss.

The solution statement as it stands juggles a lot of variables. We can rewrite the grid dimension constraints using only a,b,w,ha, b, w, h:

asw=w∧bsw≤h  ⟺  a≥whb∧sw=waas_w = w \land bs_w \le h \iff a \ge \frac{w}{h}b \land s_w = \frac{w}{a} bsh=h∧ash≤w  ⟺  b≥hwa∧sh=hb.bs_h = h \land as_h \le w \iff b \ge \frac{h}{w}a \land s_h = \frac{h}{b}.

ss is now dependent on the grid and rectangle dimensions. Using r≔whr \coloneqq \frac{w}{h}, the rectangle's aspect ratio:

aw≔min⁡{ a∈N  |  ∃b∈N . n≤ab∧a≥rb }a_w \coloneqq \min \Set{a \in \N | \exist b \in \N \text{ . } n \le ab \land a \ge rb} bh≔min⁡{ b∈N  |  ∃a∈N . n≤ab∧b≥ar }b_h \coloneqq \min \Set{b \in \N | \exist a \in \N \text{ . } n \le ab \land b \ge \frac{a}{r}} s=max⁡{ waw,hbh }.s = \max \Set{ \frac{w}{a_w}, \frac{h}{b_h}}.

awa_w is the column count of the fit-width grid, bhb_h the row count of the fit-height grid. They are minimized since the side length is inversely proportional. The side length is a trivial result of the grid dimensions; only the algorithm for finding awa_w and bhb_h remains to be developed.

Let's take a look at a fit-width grid with dimensions (aw,bw)∈N2(a_w, b_w) \in \N^2. The row count bwb_w can be abstracted away, too:

n≤awbw∧aw≥rbw  ⟺  naw≤bw≤awrn \le a_wb_w \land a_w \ge rb_w \iff \frac{n}{a_w} \le b_w \le \frac{a_w}{r}   ⟺  ⌈naw⌉≤awr\iff \left\lceil \frac{n}{a_w} \right\rceil \le \frac{a_w}{r}   ⟺  aw≥r⌈naw⌉.\iff a_w \ge r \left\lceil \frac{n}{a_w} \right\rceil.

With an analogous derivation for bhb_h, we get:

aw=min⁡{ a∈N  |  a≥r⌈na⌉ },bh=min⁡{ b∈N  |  rb≥⌈nb⌉ }.a_w = \min \Set{ a \in \N | a \ge r \left\lceil \frac{n}{a} \right\rceil }, \quad b_h = \min \Set{ b \in \N | rb \ge \left\lceil \frac{n}{b} \right\rceil }.

Since the second comparison term is non-increasing in aa (respectively in bb), it follows that every a>awa > a_w (and b>bhb > b_h) would satisfy the solution condition. The algorithm idea is therefore the following: starting from an initial guess a0≤awa_0 \le a_w (and b0≤bhb_0 \le b_h) test every value until the solution condition holds, i.e. loop while:

a<r⌈na⌉,rb<⌈nb⌉a < r \left\lceil \frac{n}{a} \right\rceil, \quad rb < \left\lceil \frac{n}{b} \right\rceil

Only the initial guesses a0a_0 and b0b_0 must be established. From the solution condition of awa_w:

aw≥r⌈naw⌉  ⟹  aw2≥rn  ⟹  aw≥rna_w \ge r \left\lceil \frac{n}{a_w} \right\rceil \implies {a_w}^2 \ge rn \implies a_w \ge \sqrt{rn}

Since aw∈Na_w \in \N it can therefore be no smaller than a0=⌈rn⌉a_0 = \left\lceil \sqrt{rn} \right\rceil. Likewise, bw≥b0=⌈nr⌉b_w \ge b_0 = \left\lceil \sqrt{\frac{n}{r}} \right\rceil. Using the loop condition above, an algorithm can finally be built: 

function fillGrid(n, w, h) {
    const r = w / h

    let a = Math.ceil(Math.sqrt(r * n))
    while (a < r * Math.ceil(n / a)) {
        a++
    }

    let b = Math.ceil(Math.sqrt(n / r))
    while (r * b < Math.ceil(n / b)) {
        b++
    }

    return Math.max(w / a, h / b)
}

Correctness is proven in the construction process above. To analyze the runtime complexity, observe that since a0≤a<r⌈na⌉a_0 \le a < r\left\lceil{\frac{n}{a}}\right\rceil, aa will be incremented at most ⌈r⌈na⌉−a0⌉\left\lceil{r\left\lceil{\frac{n}{a}}\right\rceil - a_0}\right\rceil times. Moreover, since aa increases, ⌈na⌉≤⌈na0⌉\left\lceil{\frac{n}{a}}\right\rceil \le \left\lceil{\frac{n}{a_0}}\right\rceil. Hence an upper bound on the iteration count is:

⌈r⌈na⌉−a0⌉<r⌈na0⌉−a0+1\left\lceil{r\left\lceil{\frac{n}{a}}\right\rceil - a_0}\right\rceil < r \left\lceil{\frac{n}{a_0}}\right\rceil - a_0 + 1 <rna0+r−a0+1< r \frac{n}{a_0} + r -a_0 + 1 =rn⌈rn⌉+r−⌈rn⌉+1= r\frac{n}{\left\lceil \sqrt{rn} \right\rceil} + r - \left\lceil\sqrt{rn}\right\rceil + 1 <rnrn+r−rn+1< \frac{rn}{\sqrt{rn}} + r - \sqrt{rn} + 1 =r+1= r + 1

This means at most ⌊r⌋+1\left\lfloor r \right\rfloor + 1 iterations and a runtime of O(r)\mathcal{O}(r), dependent only on the rectangle's aspect ratio. This bound is tight: for n=33,r=18.3n = 33, r = \frac{1}{8.3} the fit-width loop does 1=⌊r⌋+11 = \lfloor r \rfloor + 1 iteration, reaching the upper bound.

Lastly, geht es besser? Yes – the algorithm can be adapted to use binary search, achieving O(log⁡r)\mathcal{O}(\log r) (how? left to the reader). Geht es (noch) besser? Probably not. This problem is a particular case of integer programming: find minimal/maximal integer solution to a problem satisfying constraints. It is an NP-hard problem, with state of the art algorithms having complexity (log⁡n)O(n)⋅(m⋅log⁡V)O(1)(\log n)^{\mathcal{O}(n)} \cdot (m \cdot \log V)^{\mathcal{O}(1)}, where:

  • nn is the number of variables,
  • mm the number of constraints,
  • VV the maximum absolute value of the solution.

For our algorithm we'd solve for the fit-width and fit-height solution separately. Handling fit-width: n=1n = 1 since we just solve for awa_w, m=2m = 2 and V=rV = r since by the upper bound above a0≤a≤a0+ra_0 \le a \le a_0 + r. Plugging in the formula above, we'd get a runtime of (2log⁡r)O(1)(2\log r)^{\mathcal{O}(1)}, which is not asymptotically better than the binary search solution. The solution developed here is heuristics-based: it does a local search on the solution space, exploiting characteristics unique to this particular problem. The only way to reduce the asymptotic runtime is with a better heuristic.

We have a correct and optimal algorithm, therefore this presentation concludes here.
